<objective>
Create the `requireBiometric()` guard utility and wire it into `machine_list_screen.dart` so that both Edit and Delete taps require successful biometric/PIN authentication before proceeding.

Purpose: BIO-02 ŌĆö credentials must not be accessible to anyone who picks up an unlocked phone. The gate lives at the action callsite (list screen), not inside the edit screen, so the user never sees the form before auth.

Output: Edit and Delete taps show an OS biometric/PIN prompt; only on success does the action proceed. Auth failure cancels the action silently (OS has already shown its own error feedback).
