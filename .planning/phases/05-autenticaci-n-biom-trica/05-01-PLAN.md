## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| OS biometric subsystem → app | Biometric result comes from the OS via local_auth plugin; app trusts the bool return value |
| App cold launch → app body | LockScreen is the gate — no routes render until isAuthenticated == true |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Elevation of Privilege | AndroidManifest.xml / FlutterActivity | mitigate | FlutterFragmentActivity required; without it BiometricPrompt silently fails and attacker on device gets no auth challenge |
| T-05-02 | Elevation of Privilege | iOS Info.plist / NSFaceIDUsageDescription | mitigate | Missing key causes runtime crash; adding it per RESEARCH.md Pitfall 3 is ASVS V2 control |
| T-05-03 | Security bypass | biometricAuthProvider autoDispose | mitigate | @Riverpod(keepAlive: true) prevents state reset during GoRouter navigation; autoDispose would reset isAuthenticated between routes |
| T-05-SC | Tampering | pub.dev package install | accept | local_auth is official Flutter team package from flutter/packages monorepo — approved in RESEARCH.md Package Legitimacy Audit |
</threat_model>
