<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| OS biometric result → biometricAuthProvider | The bool from local_auth.authenticate() is the only signal that gates the app; it is OS-controlled |
| background timer → re-lock | DateTime difference check in AppLifecycleListener.onResume determines whether to re-lock |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-04 | Elevation of Privilege | AppLifecycleListener onResume | mitigate | Use DateTime diff > kLockTimeout (10 min) per BIO-03; _pausedAt reset to null after check prevents double-trigger |
| T-05-05 | Security bypass | LockScreen initState direct authenticate() call | mitigate | Wrap in addPostFrameCallback; direct call before first frame can fail silently on some platform/version combos (Assumption A1) |
| T-05-06 | Elevation of Privilege | biometricOnly: true exclusion | mitigate | Keep default (false) so OS PIN fallback is active for BIO-04; biometricOnly:true would lock out PIN-only devices with no error |
| T-05-07 | Security bypass | autoDispose on biometricAuthProvider | mitigate | Plan 01 sets keepAlive:true; this plan only reads/watches the provider and must not introduce a local autoDispose variant |
</threat_model>
