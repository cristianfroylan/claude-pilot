# Phase 6 — UI Design Contract: Session Start Picker

> Visual and interaction contract for Phase 6. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
> This is a Flutter/Material 3 app — units are dp (logical pixels) and sp (scale-independent pixels), not CSS px.

## Spacing Scale

Same 8-point scale as all prior phases. Phase 6 uses only the following:

| Token | Value | Usage in Phase 6 |
|-------|-------|-------------------|
| xs | 4 dp | Drag handle gap above sheet title; gap between path icon and path text |
| sm | 8 dp | Spacing between list items; horizontal padding inside drag handle row |
| md | 16 dp | Sheet horizontal padding; folder path list item vertical padding; form section spacing in AddEditMachineScreen |
| lg | 24 dp | Vertical gap between sheet title and folder list |
| xl | 32 dp | Sheet bottom safe area padding |

Exceptions:
- Sheet drag handle: 32 dp wide × 4 dp tall (matches VoiceBottomSheet pattern exactly)
- Folder path list items: minimum touch target 48 dp tall (Material tap target for list rows)
- Reorder drag handle icon touch target: 44 dp wide × 44 dp tall (iOS HIG minimum)

## Color

Same Material 3 dark ColorScheme from seed `#1E8BC3`. No new color values.

| Role | Material 3 Token | Approximate Hex | Usage in Phase 6 |
|------|-----------------|-----------------|------------------|
| Dominant (60%) | `colorScheme.surface` | ~#0F1117 | Picker sheet background; modal barrier |
| Secondary (30%) | `colorScheme.surfaceContainerHighest` | ~#1E2130 | Folder path row background in picker sheet; folder path item container in editor |
| Accent (10%) | `colorScheme.primary` | ~#4BA3C7 | "Choose a project" icon; active drag indicator during reorder |
| Destructive | `colorScheme.error` | ~#CF6679 | Delete folder path button (trailing icon on editor row) |

Accent (`colorScheme.primary`) reserved for ONLY these elements in Phase 6:
1. The folder icon (`Icons.folder_outlined`) in the picker sheet list header area
2. The reorder drag handle active state (when finger is holding a row mid-drag)

Accent must NOT be used on: "Start blank" button, drag handle indicator bar, individual folder path rows, the add-path text field.

The sheet drag handle bar uses `colorScheme.onSurfaceVariant.withValues(alpha: 0.4)` — same as VoiceBottomSheet.

## Component Inventory

### Component A: Folder Path Editor (inside AddEditMachineScreen)

Inserted below the Password field in `AddEditMachineScreen`, above the Save button. This is inline form content — not a separate screen or sheet.

Layout: A labeled section with a `ReorderableListView` of path rows, plus an "Add path" row at the bottom.

| Element | Spec |
|---------|------|
| Section divider | `Divider()` + `Text('Working folders', style: textTheme.bodyMedium?.copyWith(fontWeight: FontWeight.w600))` — 16 sp semibold, `colorScheme.onSurfaceVariant` color |
| Section spacing above | md (16 dp) from Password field |
| Path row | `ListTile` with `leading: Icon(Icons.drag_handle)`, `title: Text(path, style: 14sp regular)`, `trailing: IconButton(Icons.delete, color: colorScheme.error)` |
| Path row background | `colorScheme.surfaceContainerHighest`, `borderRadius: 8 dp`, margin bottom sm (8 dp) between rows |
| Drag handle icon | `Icons.drag_handle`, `colorScheme.onSurfaceVariant` color — activates `ReorderableListView` drag |
| Drag active state | Row lifts with `Material` elevation shadow; drag handle icon switches to `colorScheme.primary` |
| Add path row | `TextFormField` with `OutlineInputBorder`, label "Folder path", hint "/home/user/projects/myapp", `suffixIcon: IconButton(Icons.add)` |
| Add action | Tapping `Icons.add` or pressing `TextInputAction.done` appends the trimmed path to the list and clears the field |
| Empty state (no paths) | Text "No folders configured. Add a path to enable the session picker." — 12 sp, `colorScheme.onSurfaceVariant`, centered, 16 dp vertical padding |
| Section spacing below | lg (24 dp) from last row to Save button |

Validation:
- Path must be non-empty after trim; otherwise add is ignored silently (no error — field simply does not append)
- Duplicate paths are accepted (user responsibility)
- No path format validation (SSH server validates at `cd` execution time)

### Component B: Session Picker Bottom Sheet

Shown once after SSH connects (shell is ready) if the machine has at least one configured folder path. Shown via `showModalBottomSheet` with `isDismissible: false` and `enableDrag: false` — the only exit path is tapping a folder or "Start blank".

Layout: Non-dismissible `DraggableScrollableSheet` is NOT used. Plain modal bottom sheet with fixed structure matching VoiceBottomSheet conventions.

| Element | Spec |
|---------|------|
| Drag handle | 32 dp wide × 4 dp tall, `colorScheme.onSurfaceVariant.withValues(alpha: 0.4)`, `BorderRadius.circular(2)`, centered, sm (8 dp) top padding |
| Sheet title | "Choose a project" — 16 sp semibold, `colorScheme.onSurface`, left-aligned, md (16 dp) horizontal padding, lg (24 dp) top gap from drag handle |
| Sheet background | `colorScheme.surface` |
| Sheet horizontal padding | md (16 dp) left and right |
| Folder list | `ListView` of folder rows — each row is a full-width tappable `Material` surface |
| Folder row | `ListTile` with `leading: Icon(Icons.folder_outlined, color: colorScheme.onSurfaceVariant)`, `title: Text(path, style: 14sp regular, overflow: TextOverflow.ellipsis)`, `shape: RoundedRectangleBorder(borderRadius: 8dp)`, `tileColor: colorScheme.surfaceContainerHighest` |
| Folder row tap | Immediately pops sheet, then sends `cd <path>\n` to SSH shell (PICK-03) |
| Row spacing | sm (8 dp) between rows |
| Divider before "Start blank" | `Divider()` with vertical padding sm (8 dp) top and bottom |
| "Start blank" row | `TextButton` spanning full width, left-aligned with md (16 dp) horizontal padding, label "Start blank", style uses `colorScheme.onSurfaceVariant` foreground |
| Bottom padding | xl (32 dp) + `MediaQuery.of(context).padding.bottom` (safe area) |
| Max sheet height | 60% of screen height; if folder list overflows, list becomes scrollable inside the sheet |

Behavior rules:
- `isDismissible: false` — back gesture and barrier tap do not close the sheet. The user MUST tap a folder or "Start blank".
- `enableDrag: false` — the drag handle is decorative only (visual consistency with VoiceBottomSheet), not functional.
- Sheet never appears if `machine.folderPaths.isEmpty` (PICK-04 — checked before `showModalBottomSheet` call).

## Interaction Contract

### PICK-01: Adding a folder path

1. User types a path in the "Folder path" `TextFormField` at the bottom of the editor section
2. User taps `Icons.add` suffix icon OR presses `TextInputAction.done` on keyboard
3. If field is non-empty after trim: path appends to list above, field clears
4. If field is empty after trim: no action, no error shown, field focus retained

### PICK-02: Reordering folder paths

1. User long-presses the `Icons.drag_handle` area of any path row — drag activates
2. Row lifts with shadow elevation; drag handle icon color shifts to `colorScheme.primary`
3. User drags to desired position and releases
4. List order updates immediately; new order is saved on "Save Machine"

### PICK-02: Deleting a folder path

1. User taps `Icons.delete` (trailing icon, `colorScheme.error`) on a path row
2. Row is removed immediately from the list — no confirmation dialog (paths are trivially re-addable)
3. Change persists on "Save Machine"

### PICK-01: Picker sheet trigger (post-connect)

1. SSH session reaches shell-ready state (`SshConnected` with shell usable)
2. App checks `machine.folderPaths.isNotEmpty`
3. If true: `showModalBottomSheet` opens Component B (non-dismissible)
4. If false: session starts blank directly, no sheet (PICK-04)

### PICK-03: Selecting a folder

1. User taps a folder row in the picker sheet
2. Sheet pops immediately (`Navigator.of(context).pop()`)
3. `cd <path>\n` is written to the SSH shell stdin
4. Terminal reflects the command and working directory change

### PICK-01: Starting blank

1. User taps "Start blank" in the picker sheet
2. Sheet pops immediately
3. No command is sent to the SSH shell — session is at default shell directory

### Keyboard handling (editor section)

- When the "Folder path" field gains focus, the form scrolls to keep the field above the keyboard (handled by `SingleChildScrollView` already wrapping the form)
- `TextInputAction.done` on "Folder path" field: adds path (if non-empty) and dismisses keyboard

## Copywriting Contract

All strings in English (matches existing app convention — all prior screens are in English for UI labels, Spanish in comments/CLAUDE.md).

| Element | Copy |
|---------|------|
| Section heading (editor) | "Working folders" |
| Add path field label | "Folder path" |
| Add path field hint | "/home/user/projects/myapp" |
| Add path icon button semantics | "Add folder path" |
| Reorder handle semantics | "Reorder" |
| Delete path button semantics | "Remove folder path" |
| Editor empty state | "No folders configured. Add a path to enable the session picker." |
| Picker sheet title | "Choose a project" |
| Start blank button | "Start blank" |
| Folder row semantics suffix | "Open session in [path]" |

Tone rules (inherited from Phase 1):
- Sentence case only. No ALL CAPS, no Title Case For Every Word.
- No exclamation marks.
- No "Error:" prefix on any error state.
- Imperative verbs for CTAs: "Add", "Remove", "Start blank".

No destructive confirmation dialog for folder path deletion — paths are not credentials and are trivially re-added.

## Accessibility

- `Icons.drag_handle` row: `Semantics(label: 'Reorder', child: ...)` wraps the leading icon
- `Icons.delete` button: `Semantics(label: 'Remove folder path', child: IconButton(...))` — matches existing pattern in `machine_list_tile.dart`
- `Icons.add` suffix button: `Semantics(label: 'Add folder path', child: IconButton(...))`
- Picker sheet folder rows: `Semantics(label: 'Open session in $path', child: ListTile(...))` — path text is in the label to avoid relying on visual truncation
- "Start blank" button: label is sufficient for screen readers — no additional semantics needed
- Minimum touch targets: all list rows 48 dp tall; drag handle icon area 44 dp wide × 44 dp tall
- Color is never the sole indicator: folder rows are full-width tappable areas (shape + tileColor distinguish them from background), delete button uses icon + color + semantics label
