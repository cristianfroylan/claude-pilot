<objective>
Extend the existing InputBar command panel from a flat list of control-signal chips into a
sectioned panel with three new labeled groups of one-tap text commands: Claude slash commands,
Shell navigation, and Session commands. Each text chip sends its command string with a trailing
newline directly to the PTY via the existing `sendText` method.

This is a self-contained vertical slice: after this plan, a user can open the command panel and
execute any quick command with a single tap, with the result visible immediately in the terminal.

Purpose: Deliver CMD-01 through CMD-05 — control Claude Code without typing.
Output: An extended `input_bar.dart` with sectioned text-command chips. No new files, no new
packages, no code generation.
</objective>

| T-02-02 | Denial of Service | Rapid chip taps flooding PTY | accept | Each tap sends one short command; the remote shell rate-limits naturally. Panel-stays-open is an intentional UX decision. No mitigation needed for a single-user LAN tool. |
