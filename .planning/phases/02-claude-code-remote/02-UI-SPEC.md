## Color

All values resolved from `ColorScheme.fromSeed(seedColor: Color(0xFF1E8BC3), brightness: Brightness.dark)`.
No hardcoded hex values in new Phase 2 widgets — use `Theme.of(context).colorScheme.*` exclusively.

| Role | Token | Approximate Hex | Usage |
|------|-------|----------------|-------|
| Dominant (60%) | `colorScheme.surface` | ~#0F1117 | Terminal background (xterm TerminalTheme.background), Scaffold body |
| Secondary (30%) | `colorScheme.surfaceContainerHigh` | ~#1A1F2E | InputBar main row background, AppBar background — matches Phase 1 pattern |
| Panel surface | `colorScheme.surfaceContainerHighest` | ~#1E2430 | Expanded command panel background — matches Phase 1 pattern |
| Accent (10%) | `colorScheme.primary` | ~#4BA3C7 | Mic button active indicator dot, Approve FilledButton fill, connecting dot pulse |
| Destructive | `colorScheme.error` | ~#FF757F | Reject button text color (OutlinedButton with `styleFrom(foregroundColor: colorScheme.error)`) |
| On-surface variant | `colorScheme.onSurfaceVariant` | ~#8090A8 | Section header labels in chip panel, disabled state text |

Accent reserved for:
1. Mic button: active recording state indicator (8px dot above mic icon, same pulse animation pattern as connecting dot in Phase 1)
2. Approve button: `FilledButton` — only one per permission card — primary action fill
3. Connecting dot: already established in Phase 1, not changed

Destructive reserved for:
- Reject button foreground only — uses `OutlinedButton`, not filled, to visually distinguish from Approve

**Primary focal point:** The terminal surface (`colorScheme.surface`, xterm TerminalView) is the primary visual anchor of the screen — it occupies the majority of vertical space and carries all Claude Code output. The permission card and InputBar are secondary surfaces; they are additive chrome that does not compete with the terminal for visual weight.

## Component Inventory

### 1. InputBar — Extended Row

**Current layout (Phase 1):** `[Command toggle] [Spacer] [←][↑][↓][→]`

**Phase 2 layout:** `[Command toggle] [Spacer] [Mic IconButton] [←][↑][↓][→]`

- Mic `IconButton`: `Icons.mic`, size 20 (matches `Icons.arrow_*` at 18, slightly larger for discoverability)
- Mic button width/height: 36x36px (same as arrow buttons)
- Mic button `onPressed`: calls `_launchVoiceRecognition()` — invokes `ACTION_RECOGNIZE_SPEECH` via `android_intent_plus`
- Mic button hidden entirely (`if (_voiceAvailable)`) when voice unavailable — no placeholder, no error
- `_voiceAvailable` bool initialized at widget `initState` via a capability check; defaults `false` until confirmed

### 2. Expanded Command Panel — Sectioned Layout

**Current (Phase 1):** flat `Wrap` of control signal chips only

**Phase 2:** `Wrap` with labeled sections, vertically scrollable `SingleChildScrollView` if content exceeds viewport

```
[Section: Control]
  [Interrupt Ctrl+C]  [Exit/EOF Ctrl+D]  [Escape ESC]  [Tab]

[Section: Claude]
  [/clear]  [/compact]  [/help]  [/cost]  [/gsd]

[Section: Shell]
  [cd ~]  [cd ..]  [ls]  [pwd]

[Section: Session]
  [claude]  [claude .]  [exit]  [q]
```

- Section header widget: `Padding(padding: EdgeInsets.only(top: 8, bottom: 2), child: Text('Claude', style: TextStyle(fontSize: 11, fontWeight: FontWeight.w400, color: colorScheme.onSurfaceVariant)))`
- All chips: `ActionChip`, `label: Text(cmdLabel, style: TextStyle(fontSize: 12))`
- Wrap spacing: 8px horizontal, 4px run spacing — matches Phase 1 exactly
- Panel stays open after any tap (matches Phase 1 decision — no auto-close)
- Text commands append `\n` and call `sshSessionProvider.notifier.sendText('cmd\n')`
- Control signal chips call `sendBytes([...])` — unchanged from Phase 1

### 3. Voice Review Bottom Sheet (`ModalBottomSheet`)

Triggered by mic button result. Never auto-sends.

Layout (top to bottom inside bottom sheet):
```
[drag handle — DragHandle widget or Container 4x32px, centered, radius 2px, color onSurfaceVariant.withOpacity(0.4)]
[SizedBox height: 16]
[Text 'Review your message', style: bodyMedium weight w600, textAlign: center]
[SizedBox height: 12]
[Container — read-only transcript display]
  - color: colorScheme.surfaceContainerHighest
  - padding: 12px all sides
  - borderRadius: 8px
  - child: SelectableText(transcript, style: TextStyle(fontSize: 14, height: 1.5))
[SizedBox height: 20]
[Row children: [Expanded(Discard TextButton)], [SizedBox width: 12], [Expanded(Send message FilledButton)]]
[SizedBox height: 16 — or MediaQuery.of(context).viewInsets.bottom if keyboard open]
```

- "Send message" label: `'Send message'` — verb + noun, unambiguous action
- "Discard" label: `'Discard'` — specific to the action (discarding the transcription); not the generic 'Cancel'
- "Review your message" heading communicates what to do, not just what it is
- Bottom sheet `isScrollControlled: true` to respect keyboard insets

### 4. Permission Approval Card

Slides in above InputBar via `AnimatedSwitcher` (duration: 200ms, `FadeTransition` + `SizeTransition` combined).

Card structure:
```
Container(
  color: colorScheme.surfaceContainerHighest,
  padding: EdgeInsets.symmetric(horizontal: 12, vertical: 12),
  child: Row(
    children: [
      Icon(Icons.lock_outline, size: 16, color: colorScheme.onSurfaceVariant),
      SizedBox(width: 8),
      Expanded(
        child: Text(
          truncatedPermissionLine,  // max 80 chars + ellipsis
          style: TextStyle(fontSize: 12),
          overflow: TextOverflow.ellipsis,
        ),
      ),
      SizedBox(width: 8),
      OutlinedButton(
        onPressed: _reject,
        style: OutlinedButton.styleFrom(foregroundColor: colorScheme.error),
        child: Text('Reject'),
      ),
      SizedBox(width: 8),
      FilledButton(
        onPressed: _approve,
        child: Text('Approve'),
      ),
    ],
  ),
)
```

- Card appears between `TerminalViewWrapper` and `InputBar` in the `Column` in `TerminalScreen`
- `AnimatedSwitcher` wraps the card slot: child is the card widget when active, `SizedBox.shrink()` when inactive
- Card auto-dismisses on tap (either button); also dismisses when `permissionDetectorProvider` returns null (pattern cleared)
- No timeout auto-dismiss (deferred decision from CONTEXT.md)
- Lock icon (`Icons.lock_outline`) reinforces the security nature of the action without adding verbose text
- Approve button is FilledButton (prominent, right-side); Reject is OutlinedButton (secondary, left of Approve)
- Button labels remain `'Approve'` / `'Reject'` — the inline excerpt already provides the noun context; expanding to `'Approve action'` / `'Reject action'` adds no meaningful disambiguation and reduces label density on a narrow card

## Interaction Contract

### Voice Flow (Android only)

1. User taps mic `IconButton`
2. App calls `ACTION_RECOGNIZE_SPEECH` via `android_intent_plus` — Android system speech dialog appears (OS-managed visual, audio feedback, timeout)
3. On result: show `ModalBottomSheet` with transcript text
4. User reads transcript; taps "Send message" → `sshSessionProvider.notifier.sendText(transcript + '\n')`, sheet dismisses
5. User taps "Discard" → sheet dismisses, nothing sent
6. On `ActivityNotFoundException` or unavailability: `_voiceAvailable = false`, mic button hidden, no error shown to user

### Permission Card Flow

1. `permissionDetectorProvider` watches terminal stdout via `SshSession` interceptor
2. `permissionDetectorProvider` matches against `kPermissionPattern` regex in `permission_detector.dart`
3. On match: provider emits the last matched line (trimmed, max 80 chars)
4. `TerminalScreen` watches provider; `AnimatedSwitcher` child transitions from `SizedBox.shrink()` to card
5. User taps "Approve" → `sendBytes(utf8.encode('y\n'))`, provider resets, card slides out
6. User taps "Reject" → `sendBytes(utf8.encode('n\n'))`, provider resets, card slides out
7. Pattern clears naturally from buffer → provider emits null → card slides out automatically

### Quick Command Tap Feedback

- ActionChip provides built-in ink splash feedback — no additional toast or SnackBar
- Terminal echo of the command confirms it was sent (visible immediately in TerminalView)
- Panel stays open for rapid successive commands

### Animations

| Widget | Animation | Duration | Curve |
|--------|-----------|----------|-------|
| Permission card slide-in | `AnimatedSwitcher` with `SizeTransition` + `FadeTransition` | 200ms | `Curves.easeInOut` |
| Permission card slide-out | same (reversed) | 200ms | `Curves.easeInOut` |
| Command panel expand | `AnimatedSwitcher` (already Phase 1 — `setState` toggle) | inherit Flutter default (~300ms) | default |
| Connecting dot pulse | `AnimationController.repeat(reverse: true)` 1.0 ↔ 0.4 opacity | 800ms | linear (Phase 1, unchanged) |

Destructive actions in this phase:
- "Reject" on permission card: sends `n\n` to PTY — this is a Claude Code permission rejection, not a Flutter destructive action. No confirmation dialog. Single tap sends immediately (same model as control signal chips). The `OutlinedButton` with error color is the visual signal.
- "Discard" on bottom sheet: discards the voice transcription with no undo. No confirmation dialog — the bottom sheet itself is the review step.
- No data deletion in this phase.

## Accessibility

- Mic button: `Semantics(label: 'Start voice input', child: IconButton(...))`
- Approve button: standard `FilledButton` — screen reader reads label "Approve"
- Reject button: standard `OutlinedButton` — screen reader reads label "Reject"
- Permission card icon: `Icon(..., semanticLabel: null)` — decorative, label provided by surrounding text
- Bottom sheet: `isScrollControlled: true` — keyboard insets respected via `MediaQuery.of(context).viewInsets.bottom` padding
- ActionChip labels are descriptive (include key combo in control signals: "Interrupt [Ctrl+C]") — unchanged from Phase 1
