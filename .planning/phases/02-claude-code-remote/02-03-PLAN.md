<objective>
Add voice dictation: a mic button in the InputBar that uses `speech_to_text` to capture spoken
input, presents the final transcript in a review bottom sheet, and sends it to the PTY only when
the user explicitly taps Send. When the device has no speech service, the mic button is hidden
with no error (VOZ-04).

This is a vertical slice: after this plan, a user can tap the mic, speak a prompt, review the
transcription, and send it to Claude Code — all without typing.

Critical finding (RESEARCH.md): the CONTEXT.md `android_intent_plus` decision CANNOT be
implemented as written — `android_intent_plus` has no `launchForResult` and cannot receive the
transcription. The locked decision says "or equivalent"; the equivalent is `speech_to_text`
v7.4.0 (already documented in CLAUDE.md as the chosen voice package). This plan uses
`speech_to_text`, not `android_intent_plus`.

Purpose: Deliver VOZ-01..04 — dictate prompts by voice with explicit review-before-send.
Output: pubspec + manifest changes, a new VoiceBottomSheet widget, and mic wiring in InputBar.
Depends on Plan 02-01 because both modify `input_bar.dart` (sequential to avoid conflicts).

NOTE: `speech_to_text` is tagged `[ASSUMED]` in the RESEARCH Package Legitimacy Audit (slopcheck
unavailable). Task 1 is a blocking human-verify checkpoint that must pass before `flutter pub add`.

speech_to_text 7.4.0 API (RESEARCH.md Pattern 3 + assumption A2):
```dart
import 'package:speech_to_text/speech_to_text.dart';
final _speech = SpeechToText();
final bool available = await _speech.initialize(onStatus: (_) {}, onError: (_) {});
await _speech.listen(
  onResult: (result) {
    // result.finalResult (bool), result.recognizedWords (String)
  },
  listenFor: const Duration(seconds: 30),
  pauseFor: const Duration(seconds: 5),
  localeId: null, // device default (Open Question 1 — Claude's Discretion)
);
_speech.stop();   // stop listening
_speech.cancel(); // in dispose()
// If `listen(listenFor:/pauseFor:)` top-level params fail to compile (A2),

```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Microphone → app | Spoken audio captured by the OS speech recognizer, returned as transcript text |
| Transcript → PTY write | User-authored transcript sent to the user's own authenticated SSH session after explicit review |
| npm/pub install | `speech_to_text` package fetched from pub.dev |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-SC | Tampering | pub.dev install of speech_to_text ([ASSUMED]) | mitigate | Blocking human-verify checkpoint (Task 1) confirms publisher csdcorp.com + pub.dev legitimacy before `flutter pub add`. Never auto-approvable. |
| T-02-07 | Tampering | Voice transcript injected into PTY | accept | Per RESEARCH Security Domain V5: the user reviews the transcript in the bottom sheet and must tap Send explicitly. Same threat model as typed input to one's own machine. |
| T-02-08 | Information Disclosure | Microphone access (RECORD_AUDIO) | mitigate | Permission declared in manifest and requested at runtime via the OS dialog; capture is OS-managed and only triggered by an explicit mic tap. No audio is stored or transmitted off-device by the app. |
| T-02-09 | Information Disclosure | initialize() false leaks no error (VOZ-04) | accept | Intentional: failure self-hides the mic button silently — no information disclosed, correct graceful-degradation behavior. |
</threat_model>
