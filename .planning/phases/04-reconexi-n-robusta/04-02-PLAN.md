    Countdown helper `_waitWithCountdown(int seconds, {required bool isMidSession, required int attempt, required int maxAttempts})`: cancel any existing `_countdownTimer` FIRST (Pitfall 2). After the wait, try `_connectOnce(...)`; on success install a fresh done-watcher and `state = AsyncData(SshConnected(_terminal!))`; return. On failure continue the loop. After loop exhausts, `state = AsyncData(SshFailed(_terminal!))`.

    Mid-session drop detection (Pattern 5 + Pitfall 5): replace the current `_client!.done.catchError(... AsyncError ...)`. After a successful `_connectOnce`, increment `_connectionGeneration` to a captured local `gen`, and attach `_client!.done.then((_) { if (!_disposed && _isMidSession && gen == _connectionGeneration) _runMidSessionRetry(); }, onError: (Object _) { if (!_disposed && _isMidSession && gen == _connectionGeneration) _runMidSessionRetry(); });`. The generation guard prevents a stale done-callback from a replaced client firing against the new connection.

    Public `reconnect()` (RECON-04 — single attempt per spec "attempt one more connection manually"): guard `if (_disposed) return;`. Set `_cancelRequested = false`. Then `try { await _connectOnce(...); install done-watcher; _isMidSession = true; state = AsyncData(SshConnected(_terminal!)); } catch (_) { state = AsyncData(SshFailed(_terminal!)); }`. Read machine+password the same way `build()` does. Do NOT call `ref.invalidateSelf()` (Pitfall 1 — would recreate Terminal and lose scrollback).
