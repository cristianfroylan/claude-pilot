      comma that ends the `body:` argument).

      The resulting structure:

          top: true,

          left: false,

          child: Column(
            children: [
              Expanded(...),
              AnimatedSwitcher(...),
              InputBar(machineId: machineId),
            ],
          ),
        ),

      Use SafeArea with top: true only. Do NOT wrap the Scaffold itself — the AppBar
      handles status-bar inset independently. Wrapping the Scaffold adds double padding.

    EDIT B — Add MediaQuery viewInsets dependency + ValueKey in the data: branch of
    sessionAsync.when() (per CONTEXT.md locked decision and RESEARCH.md Pattern 2):

      In the Expanded child's `data:` lambda, before returning TerminalViewWrapper,
      read viewInsets.bottom to register a MediaQuery rebuild dependency, then pass
      a ValueKey keyed to that height:

        data: (terminal) {
          final keyboardHeight = MediaQuery.of(context).viewInsets.bottom;
          return TerminalViewWrapper(
            key: ValueKey(keyboardHeight),
            machineId: machineId,
            terminal: terminal,
          );
        },

      Reading `MediaQuery.of(context).viewInsets.bottom` in the build method creates
      a reactive dependency: when the keyboard appears/disappears or the device rotates,
      Flutter rebuilds this widget, TerminalViewWrapper receives a new ValueKey, and
      LayoutBuilder fires with updated constraints. This is in addition to the existing
      `resizeToAvoidBottomInset: true` on the Scaffold (do not remove that).

      Note: the `data:` callback currently uses `=>` (arrow syntax). Change it to a
      block body `{ ... }` to accommodate the `final keyboardHeight =` statement.

    No new imports needed — MediaQuery is in package:flutter/material.dart, already
    imported.
  </action>
