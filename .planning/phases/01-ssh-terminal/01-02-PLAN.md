<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user input → storage | Machine form fields (name, host, port, username, password) cross into persistent storage |
| device storage → app | flutter_secure_storage reads OS-encrypted credential store on app launch |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Information Disclosure | SSH password storage | mitigate | Password stored ONLY in flutter_secure_storage (`ssh_password_<uuid>`), never in shared_preferences; enforced by MachineRepository design |
| T-02-02 | Information Disclosure | Password field UI | mitigate | `obscureText: true` on password TextField; visibility toggle available but not on by default; password never logged |
| T-02-03 | Tampering | Machine list data integrity | accept | shared_preferences data is not sensitive (no passwords); tampering would only affect non-credential metadata; low impact |
| T-02-04 | Denial of Service | Invalid port input | mitigate | Port field validator enforces int 1–65535; invalid input blocks form submission |
| T-02-05 | Information Disclosure | Credential backup | mitigate | android:allowBackup="false" set in Plan 01; flutter_secure_storage keys are device-bound — non-exportable |
| T-02-SC | Tampering | pub.dev package installs | accept | All packages verified in RESEARCH.md; no new packages in Plan 02 beyond those already installed |
</threat_model>
