## Decisions Made

- **Provider class naming:** Used `MachineNotifier` (not `MachineRepository`) to avoid a name collision with the repository class in the same build unit. The generated provider variable is `machineProvider` (riverpod_generator 4.0.3 drops the "Notifier" suffix).

- **`state.value` instead of `state.valueOrNull`:** riverpod 3.2.1 (the installed version) exposes `.value` as the nullable getter on `AsyncValue`. The `valueOrNull` convenience getter was added in a later Riverpod version — not available in the pinned version from Plan 01.

- **Timestamp-based machine id:** `DateTime.now().millisecondsSinceEpoch.toString()` used in both `Machine.generate()` and `AddEditMachineScreen._save()`. Sufficient for a list of <10 machines with no concurrent creation; no `uuid` package needed.
