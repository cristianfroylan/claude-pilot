<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| phone → SSH server (LAN) | SSH handshake + encrypted session; password transmitted over dartssh2 SSH transport |
| SSH stdout → terminal widget | Raw bytes from server process written to xterm Terminal; no sanitization needed (xterm handles VT100) |
| InputBar → SSH stdin | User text and raw control bytes transmitted to remote shell |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Information Disclosure | SSH password in memory | accept | Password loaded from flutter_secure_storage into memory only during connection; not logged, not stored in variables that persist after connection completes; Dart GC handles cleanup |
| T-03-02 | Spoofing | SSH host key verification | accept (v1) | dartssh2 does not enforce host key verification by default; v1 is LAN-only on a trusted home network; host key pinning is v2 hardening |
| T-03-03 | Denial of Service | Uncaught SSHStateError crash | mitigate | `_client!.done.catchError()` at connection time funnels transport errors to `state = AsyncError()` — app shows SnackBar instead of crashing |
| T-03-04 | Tampering | Raw bytes via control chips | accept | Ctrl+C (0x03), Ctrl+D (0x04), ESC (0x1b) are standard terminal control sequences; they cannot escalate privileges beyond what the SSH session already allows |
| T-03-05 | Information Disclosure | Terminal output visible on screen | accept | User is viewing their own remote session; no third-party screen access threat in LAN-only scope |
| T-03-06 | Denial of Service | SSH session leak on pop | mitigate | `ref.onDispose(() { _session?.close(); _client?.close(); })` in AsyncNotifier guarantees cleanup when TerminalScreen is popped; null-safe calls prevent throw on already-closed connection |
| T-03-SC | Tampering | pub.dev package installs | accept | No new packages in Plan 03; all packages verified in RESEARCH.md audit |
</threat_model>
