## Spacing Scale

Declared values (multiples of 4, in dp):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4 dp | Icon-to-label gaps, chip internal padding |
| sm | 8 dp | Compact element spacing, form field gaps |
| md | 16 dp | Default element spacing, card padding, AppBar horizontal padding |
| lg | 24 dp | Section padding, FAB bottom offset |
| xl | 32 dp | Layout gaps between major sections |
| 2xl | 48 dp | Not used in Phase 1 |
| 3xl | 64 dp | Not used in Phase 1 |

Exceptions:
- Touch targets (Ctrl+C, Ctrl+D, ESC chips, send IconButton): minimum 44 dp tall to meet iOS HIG / Android material touch target guidelines
- InputBar bottom padding: `MediaQuery.viewInsets.bottom` driven — no fixed value
- TerminalView padding: 0 dp on all sides (terminal content fills its Expanded slot edge-to-edge)

## Typography

All type is Flutter Material 3 TextTheme. Declare sizes in sp (Flutter scale-independent pixels, equivalent to dp for type).

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body (UI) | 14 sp | 400 (regular) | 1.43 (Material default) | Machine list items, form field values, status labels |
| Label | 12 sp | 500 (medium) | 1.33 | Chip labels (Ctrl+C, Ctrl+D, ESC), AppBar subtitle |
| Heading | 16 sp | 600 (semibold) | 1.5 | AppBar title (machine name), section headers in add/edit form |
| Terminal | xterm.dart default (~14 sp equivalent) | monospace regular | 1.0 (terminal line height) | All terminal output — executor must NOT override |

Typography rules:
- Maximum 3 UI type sizes in Phase 1: 12 sp, 14 sp, 16 sp. Terminal type is separate and xterm-controlled.
- Do not add a Display size — no marketing/hero text in Phase 1.
- `textScaleFactor` clamped to max 1.3 on the terminal screen to prevent layout overflow. Apply via `MediaQuery` wrapper on `TerminalScreen`.

## Color

Material 3 dark `ColorScheme` generated from seed color `#1E8BC3` (terminal-blue). Apply via `ColorScheme.fromSeed(seedColor: Color(0xFF1E8BC3), brightness: Brightness.dark)`.

| Role | Material 3 Token | Approximate Hex | Usage |
|------|-----------------|-----------------|-------|
| Dominant (60%) | `colorScheme.surface` | ~#0F1117 | App background, Scaffold background, terminal background |
| Secondary (30%) | `colorScheme.surfaceContainerHigh` | ~#1A1D24 | Machine list cards, add/edit form card, AppBar surface |
| Accent (10%) | `colorScheme.primary` | ~#4BA3C7 | See reserved-for list below |
| Destructive | `colorScheme.error` | ~#CF6679 | Delete machine action only |

Accent (`colorScheme.primary`) is reserved for ONLY these elements:
1. FAB (add machine button) background
2. Send button icon
3. Active/connected status dot
4. Focused `TextField` border and cursor color

Accent must NOT be used on: chip buttons (Ctrl+C, Ctrl+D, ESC), AppBar, inactive states, body text.

Connection status colors (semantic overrides — do not use `colorScheme.primary` for these):
- Connected: `Colors.green.shade400` (#66BB6A)
- Disconnected: `colorScheme.onSurfaceVariant` (muted grey)
- Error: `colorScheme.error` (#CF6679)
- Connecting: `colorScheme.secondary` (animated — see Interaction section)

Source: CONTEXT.md `## Implementation Decisions > Machine Manager UI` (colored dot + text label pattern locked)

## Screen Inventory

### Screen 1: Machine List (`/machines`)

Layout: `Scaffold` with `AppBar` + `ListView` + `FloatingActionButton`.

| Element | Spec |
|---------|------|
| AppBar title | "Machines" — 16 sp semibold |
| AppBar background | `colorScheme.surfaceContainerHigh` |
| Machine list item | `ListTile` — leading = status dot (12 dp circle), title = machine name (14 sp regular), subtitle = `user@ip:port` (12 sp, `onSurfaceVariant`) |
| Status dot | 12 dp diameter `Container` with `BoxDecoration(shape: BoxShape.circle)` |
| FAB | `FloatingActionButton` with `Icons.add`, positioned bottom-right, offset lg (24 dp) from edges |
| Empty state | Centered column — heading "No machines yet" (16 sp semibold), body "Tap + to add your first machine" (14 sp, `onSurfaceVariant`) |
| Swipe-to-delete | `Dismissible` widget — swipe left reveals red destructive background with `Icons.delete` icon |

### Screen 2: Add / Edit Machine (`/machines/add`, `/machines/:id/edit`)

Layout: `Scaffold` with `AppBar` + `SingleChildScrollView` containing a `Column` of form fields.

| Element | Spec |
|---------|------|
| AppBar title | "Add Machine" / "Edit Machine" — 16 sp semibold |
| AppBar leading | `Icons.arrow_back` (back navigation) |
| AppBar actions (edit only) | `Icons.delete` — destructive, `colorScheme.error` tint |
| Form fields | `TextFormField` with `OutlineInputBorder` — fields: Name, Host/IP, Port (default "22"), Username, Password |
| Password field | `obscureText: true` with toggle `Icons.visibility` / `Icons.visibility_off` |
| Field spacing | sm (8 dp) between label and field, md (16 dp) between fields |
| Save button | Full-width `FilledButton` at bottom — label "Save Machine" |
| Form padding | md (16 dp) horizontal, md (16 dp) vertical |

### Screen 3: Terminal (`/terminal/:machineId`)

Layout: `Scaffold` — `resizeToAvoidBottomInset: true` — `Column` with `Expanded(TerminalView)` + `InputBar`.

| Element | Spec |
|---------|------|
| AppBar | Minimal — title = machine name (16 sp semibold) + subtitle = connection status label (12 sp); trailing `Icons.close` for disconnect |
| AppBar background | `colorScheme.surfaceContainerHigh` |
| TerminalView | `Expanded`, fills remaining height edge-to-edge, background matches `colorScheme.surface` (xterm theme set accordingly) |
| InputBar | Pinned above keyboard — `Column`: top row = control chips (`Ctrl+C`, `Ctrl+D`, `ESC`), bottom row = `TextField` + send `IconButton` |
| Control chips | `ActionChip` or `InputChip` — label 12 sp medium, min height 44 dp, `colorScheme.surfaceContainerHigh` background |
| TextField | Single-line, `TextInputAction.send`, hint "Type a prompt…", 14 sp regular |
| Send button | `IconButton` with `Icons.send`, `colorScheme.primary` color, min size 44 dp |
| InputBar background | `colorScheme.surfaceContainerHigh` |
| InputBar padding | sm (8 dp) all sides |

## Interaction Contract

### Connection States (SSH-02)

The terminal AppBar subtitle + status dot reflect connection state at all times. States and their visual treatment:

| State | Dot color | Subtitle text | Additional UI |
|-------|-----------|---------------|---------------|
| Connecting | `colorScheme.secondary` (pulsing opacity animation 1.0→0.4, 800ms repeat) | "Connecting…" | InputBar disabled (chips + send button `onPressed: null`) |
| Connected | `Colors.green.shade400` | "Connected" | InputBar fully enabled |
| Error | `colorScheme.error` | "Connection failed" | `SnackBar` with error message (see Copywriting) |
| Disconnected | `onSurfaceVariant` | "Disconnected" | InputBar disabled |

### Delete Machine Confirmation

A confirmation dialog appears before deleting a machine (my discretion per CONTEXT.md).

- Trigger: Swipe-to-delete on list OR delete `IconButton` in edit screen
- Dialog: `AlertDialog` — title "Delete machine?", content "This will remove [machine name] and its saved credentials. This cannot be undone.", confirm button "Delete" (`colorScheme.error`), cancel button "Cancel"
- No confirmation on swipe undo (Dismissible provides undo snackbar instead)

### PTY Resize (SSH-04)

- `LayoutBuilder` wraps the `TerminalView`
- On constraint change: recalculate `columns = width ~/ 8`, `rows = height ~/ 16`, call `session.resizeTerminal(columns, rows)`
- Initial size: 80 columns × 24 rows until first layout measurement

### Keyboard Dismiss

- Tapping outside the TextField: keyboard dismisses (standard Flutter behavior)
- Android back gesture: dismisses keyboard if open; does not disconnect (disconnect is explicit AppBar action only)

## Accessibility Minimums

- All interactive elements: `Semantics` label where icon-only (send button: `"Send"`, disconnect: `"Disconnect"`, delete: `"Delete machine"`)
- Minimum touch target: 44 dp (iOS HIG) applied via `ConstrainedBox` where needed
- Status dot color is NOT the only indicator of state — text label always accompanies it (WCAG 1.4.1 Use of Color)
- TerminalView: `ExcludeSemantics(child: TerminalView(...))` — raw terminal output is not useful to a screen reader and produces noise
