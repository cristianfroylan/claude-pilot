## Accomplishments

- SshSession AsyncNotifier.autoDispose.family: SSHClient connected via SSHSocket, PTY shell `xterm-256color`, stdout/stderr piped through `Utf8Decoder(allowMalformed: true)` to `terminal.write`
- Crash-safe disconnect: `_client!.done.catchError` funnels network drops to `state = AsyncError`, triggers SnackBar, no unhandled SSHStateError
- PTY resize: TerminalViewWrapper LayoutBuilder computes `cols = width ~/ 8` (clamp 40..220) and `rows = height ~/ 16` (clamp 10..60), queued via `addPostFrameCallback`
- InputBar: Ctrl+C (0x03), Ctrl+D (0x04), ESC (0x1b) as raw bytes; TextField with TextInputAction.send; all controls disable when `!sessionAsync.hasValue`
- TerminalScreen: AppBar shows machine name + connection status + animated pulsing dot during connecting; `resizeToAvoidBottomInset: true`; TextScaler clamped to 1.3
- `flutter analyze lib/` zero errors; `flutter build apk --debug` succeeds

## Decisions Made

- **`_disposed` flag instead of `mounted`:** riverpod_generator 4.0.3 generates `_$SshSession extends $AsyncNotifier<Terminal>` — the `mounted` getter from older Riverpod versions is not available on `$AsyncNotifier`. Replaced with a `bool _disposed = false` field set in `ref.onDispose`.

- **Main branch merge at execution start:** Worktree branch `worktree-agent-aa932121ee077b6e9` was created before Plans 01 and 02 landed on main. Required `git merge main --no-edit` to bring in the Flutter project. Same pattern as Plan 02 execution.

- **`.value` instead of `.valueOrNull`:** `AsyncValue.valueOrNull` does not exist in riverpod 3.2.1 (pinned from Plan 01). Used `ref.watch(machineProvider).value` (nullable `T?`).

- **AnimationController for connecting dot:** `AnimatedOpacity` has no `repeat` parameter. Implemented `_ConnectingDot` as a `StatefulWidget` with `AnimationController.repeat(reverse: true)` oscillating opacity 1.0 ↔ 0.4 over 800ms, matching the UI-SPEC interaction contract.
